    List of Elements

    This page shows every element available and its representation using JSON, to see a list of available functio go to documentation or go to here to see a list of examples.

    Repository

    A repository is the main element in Repositorium, it represents a community of users and a set of documents related to that community

    properties
    Property Description
    id It's the identifier of a repository, you'll be using this in almost every request
    name Name of this repository.
    description A brief description of this repository
    author the author of this repository, this is an user element
    active Indicates whether this repository is active or not. Posibles values are 0 for inactive and 1 for active
    Example "id":"20","name":"Fondue","description":"This repository contains stimations of nutricional facts for food.","author":user,"active":"1"}

    User

    The User element contains the basis information about user

    properties
    Property Description
    id It's the identifier of a user
    email User email, this is also the username for login
    name User name
    lastaname User lastname
    created Indicates the date when user joined Repositorium
    active Indicates whether this user is active or not. Posibles values are 0 for inactive and 1 for active
    Example {"id":"9","email":"cgajardo@litmon.com","name":"carlos","lastname":"gajardo","created":"2011-09-25 20:26:51","active":"1"}

    Challenge

    This element represents a Repositorium's Challenge

    properties
    Property Description
    id It's the identifier of the challenge. This ID must be providen when submiting an answer.
    question The question that must be answered regarding the documents attached
    documents Array of documents for which the question must be answered.
    answera Option A for answering the question. User must decide between this option, or B
    answerb Option B for answering the question. User must decide between this option, or A
    Example {"id":"1f0e3dad9","question":"Es esto un alimento?","documents":[documents],"answera":"No","answerb":"Yes"}

    Document

    The Document element represents a document from a repository

    properties
    Property Description
    id It's the identifier of the documents. Real id is only available for admin users, other users will recieve a faked id to use when asnswering challenges.
    title The title of the document.
    content Text content of the document.
    author An user element representing the author of the document.
    created Date when the document was created.
    files Array of files attached to the document.
    tags Array of tags that describe the document.
    Example {"id":"274","title":"manzana","content":"Awesome fruit","author":user,"created":"2012-05-10 11:35:37","files":[],"tags":["fruta","casera"]}

    Stat

    This element represents the stats of a repository

    properties
    Property Description
    users Number os users participating on geiven a repository.
    documents Number of documents in a given repository.
    criterion Number of criterion in a given repository.
    Example {"users":"8","documents":"25","criterion":"3"}

    Error

    This element is used for reporting Server Errors (5XX), Client Errors (4XX) or Success (2XX)

    properties
    Property Description
    status Status code and name.
    message Brief explanation of the error.
    Example {"status":"401 Unauthorized","message":"User must be logged in"}

    Authors and Contributors

    You can @mention a GitHub username to generate a link to their profile. The resulting <a> element will link to the contributor's GitHub Profile. For example: In 2007, Chris Wanstrath ( @defunkt), PJ Hyett (@pjhyett), and Tom Preston-Werner (@mojombo) founded GitHub.

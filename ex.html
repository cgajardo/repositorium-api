Resource List all Repositories
Method GET
URL /repositories
Params -
Call example String url = "http://repositorium.cl/api/v1.0/repositories";
HttpGet httpGet = new HttpGet(url);
HTTPResponse reponse = httpClient.execute(httpGet);
String responseBody = EntityUtils.toString(response.getEntity());
Response example responseBody should look like this:
{"id":"20","name":"Fondue","description":"This repository contains stimations of nutricional facts for food.","author":{"email":"cgajardo@litmon.com","name":"carlos","lastname":"gajardo","created":"2011-09-25 20:26:51","active":"1"},"active":"1"},{"id":"21","name":"benchmark","description":"benchmark for carlos engineering thesis","author":{"email":"cgajardo@litmon.com","name":"carlos","lastname":"gajardo","created":"2011-09-25 20:26:51","active":"1"},"active":"1"}]

Resource Search on a Repository
Method GET
URL /repositories/_id/search:_query
Params fields:tags|title|content
Call example String url = "http://repositorium.cl/api/v1.0/repositories/20/search:manzana";
String params = "fields=title"; HttpGet httpGet = new HttpGet(url+"?"+params);
HTTPResponse reponse = httpClient.execute(httpGet);
String responseBody = EntityUtils.toString(response.getEntity());
Response example responseBody would depend on weather users has enough points to download a document or not. On the first case it will return a document:
{"title":"manzana","content":"Awesome fruit","author":{"email":"cgajardo@litmon.com","name":"carlos","lastname":"gajardo","created":"2011-09-25 20:26:51","active":"1"},"created":"2012-05-10 11:35:37","files":[],"tags":["fruta","casera"]}
On the second case, it will return a challenge:
{"id":"1f0e3dad9","question":"Es esto un alimento?","documents":[{"id":1,"title":"pizza","content":"Super awesome meal","author":{"email":"cgajardo@litmon.com","name":"carlos","lastname":"gajardo","created":"2011-09-25 20:26:51","active":"1"},"created":"2012-07-09 00:59:17","files":[],"tags":[]}],"answera":"No","answerb":"Yes"}
Resource Load a document
Method GET
URL /repositories/_id/documents
Params id:_id
Call example String url = "http://repositorium.cl/api/v1.0/repositories/20/documents";
String params = "id=285"; HttpGet httpGet = new HttpGet(url+"?"+params);
HTTPResponse reponse = httpClient.execute(httpGet);
String responseBody = EntityUtils.toString(response.getEntity());
Response example responseBody would depend on weather user has rol admin or not. On the first case it will return a document:
{"id":"285","title":"frambuesa","content":"No so awesome fruit","author":{"email":"cgajardo@litmon.com","name":"carlos","lastname":"gajardo","created":"2011-09-25 20:26:51","active":"1"},"created":"2012-05-10 11:49:06","files":[],"tags":["fruta"]}
On the second case, it will return with and error:
{"status":"401 Unauthorized","message":"This action can only be performed by admin users"}
